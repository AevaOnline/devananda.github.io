Havana summit

(April 2013)

Features for Bare Metal increasingly don't fit in Nova. At the Havana design summit, I proposed to split bare metal out of Nova. About a month later, took it to a vote by the TC, and it passed. Thus, Ironic was born.

OpenStack Ironic

Not just a split - a redesign

Ironic is a service to
provision
bare metal nodes

Ironic is not...

  • a Configuration Management Database
  • a hardware lifecycle management tool

Project Goals

  • Be a hardware API and no more
  • Be resilient to failures
  • Don't re-invent the wheel
  • Be extensible
  • Admins use Ironic API to enroll, configure
  • Tenants use Nova API for boot, destroy, restart, etc
  • Network config done by Neutron (*)
  • Storage config done by Cinder (*)

some significant limitations

Networking

DHCP via dnsmasq. Non HA deploy nodes. Static IP injected. Work in Neutron...

disk / raid configuration

Plans to implement this via Cinder volume specification. No work done yet.

Firmware security

Tenant has full root on host, can do *anything* to the box. Don't reuse boxes. Don't allow untrusted tenants.

Other considerations

Flavor vs hardware spec. Quotas and whole-host allocation. Billing. Mixed bm & virt clouds.
